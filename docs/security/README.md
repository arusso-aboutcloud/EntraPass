# Security Stack — Aboutcloud / EntraPass

This directory documents the Cloudflare security configuration protecting
`aboutcloud.io` and all Aboutcloud sub-properties including `entrapass.aboutcloud.io`.

## Files

| File | Description |
|---|---|
| [`cloudflare-rules.md`](cloudflare-rules.md) | Human-readable rule documentation (auto-generated) |
| [`cf-rules-dump.json`](cf-rules-dump.json) | Raw Cloudflare API export (auto-generated) |

> Both files are generated by [`infra/export-cf-rules.ps1`](../../infra/export-cf-rules.ps1).
> Re-run after any rule change to keep this in sync.

## GitHub Actions secrets required

| Secret | Purpose |
|---|---|
| `CLOUDFLARE_API_TOKEN` | Zone WAF + Transform Edit permissions |
| `CF_ADMIN_IP` | Admin home IP — redacted as `[ADMIN-IP]` in committed docs |

## Workflows

| Workflow | Trigger | What it does |
|---|---|---|
| `cf-apply.yml` | Manual | Applies hardening (WAF + headers), then exports clean docs |
| `cf-export.yml` | Manual | Read-only export of current rules to `cloudflare-rules.md` |

## How to apply hardening

Trigger **Actions → Apply Cloudflare Hardening → Run workflow**.
The workflow applies WAF + header changes to Cloudflare, then commits
a redacted `cloudflare-rules.md` as the new source of truth.

## How to regenerate docs only

```powershell
# From repo root — set your token and admin IP first (never commit these)
$env:CF_API_TOKEN = "your-token-here"
$env:CF_ADMIN_IP  = "your-home-ip"

.\infra\export-cf-rules.ps1 -RedactIp $env:CF_ADMIN_IP
```

Then commit the updated `cloudflare-rules.md` (not `cf-rules-dump.json` — gitignored).

## Architecture overview

```
Internet
   │
   ▼
Cloudflare Edge (aboutcloud.io zone)
   │
   ├─ WAF Custom Rules (5/5)       ← evaluated first, in order
   │    1. Allow My IP (bypass all checks)
   │    2. Block malicious / scanner user-agents (global)
   │    3. Block admin paths from non-admin IPs (global)
   │    4. EntraRoleLens — block API abuse and path probing
   │    5. Umami — protect analytics dashboard
   │
   ├─ Rate Limiting Rules (1/1)
   │    1. Blog API rate limit (api.aboutcloud.io)
   │
   ├─ Transform / Redirect / Config Rules
   │    - Redirect apex → blog.aboutcloud.io
   │    - Origin rule: www.aboutcloud.io → port 443
   │    - Configuration rules: Ghost admin, MTA-STS bypass
   │
   ├─ Response Header Transform Rules (1/10)
   │    1. Blog security headers (aboutcloud.io/ghost/)
   │    [PLANNED] EntraPass security headers (entrapass.aboutcloud.io)
   │
   └─ Cloudflare Pages / Workers
        ├─ entrapass.aboutcloud.io  → EntraPass (Pages)
        ├─ entraerrors.aboutcloud.io
        ├─ entratracker.aboutcloud.io
        └─ entrarolelens.aboutcloud.io
```

## Access policy

| Source | Access |
|---|---|
| Admin IP (`[ADMIN-IP]`) | Full bypass (Rule 1 Skip) — can access all paths and *.pages.dev preview URLs |
| All other IPs | *.pages.dev and *.workers.dev URLs blocked (force custom domain) |
| Scanner user-agents | Blocked globally on all properties |
| Non-admin IPs | Admin/sensitive paths (`.env`, `.git`, `/admin`, `/wp-*`) blocked |
